Hypertext Markup Language

First developed by Tim Berners-Lee in 1990; HTML stands for Hypertext Markup Language. Where Hypertext means “text with links in it” and a Markup Language is a programming language used to make text do more than just sit on a page: it can turn text into images, links, tables, lists, and much more. HTML can be seen as the 'skeleton that gives every web page structure'. Like any language, it has its own special syntax (rules for communicating).

Internet browser’s, such as Chrome, Firefox, Internet Explorer, transform the HTML code into a recognizable webpage. However you must command the browser to do so:

  • Always put <!DOCTYPE html> on the first line. This tells the browser what language it's reading (in this case, HTML).
  • Always put <html> on the next line to start the HTML document.
  • Always put </html> on the last line to end the HTML document.

Everything in your HTML file will go between your two html tags.

Tags

Things inside <>s are called tags. Tags nearly always come in pairs: an opening tag (<tag>) and a closing tag (</tag>). Whenever you open a tag, you must also close it. Tags also nest, so the most recently opened tag should be the first one closed:

 <firsttag><secondtag> Your Text </secondtag> </firsttag>

Two Parts of a HTML File

Inside your <html> tags; there is the Head and the Body. The Head contains information about your HTML file, such as it's links and title. Whereas the Body contains the majority of your visible content:

<html>
  	<head>
 	 		<title></title>
  	</head>
 	<body>
 		<p></p>
 	</body>
</html>

Paragraph Tags

The HTML paragraph element is used to hold one or more sentences. Each paragraph starts with an opening <p> tag, contains written content in the middle, and ends with a closing </p> tag:

 <p>Paragraph Text Here</p>

Heading Tags

The HTML Headings element contains emphasized text. There are six heading elements: h1, h2, h3, h4, h5, and h6. h1 being the largest heading and h6 being the smallest. 

 <h1>Heading Text Here</h1>

HTML Hyperlinks

You can link your website content to another source using Hyperlinks (links). You link to a website in HTML like this: 

 <a href="https://www.codecademy.com">Description of link </a>

This would show: Description of link. Remember URL (Web Adresses) are formated like this:

HTML Images

To insert an image you tell the tag where to get the picture using src (source code). Every image on the web has its own image URL. Simply right-click on an image and choose "Copy image URL." Paste that URL in quotes after src= to insert it with your <img> tag:

<img src="https://imageURL">

HTML Videos

You insert an HTML video much like you would insert an HTML image, however Videos have many more attributes associated with them:

  • Width and height: Set the size of the screen that displays the video.
  • Controls: Adds play, pause and volume control.
  • Source src: Sets the URL of the video to play.
  • Type: Specifies different video formats.

<video width="320" height="240" controls> <source src="video-url.mp4" type="video/mp4"> </video>

Ordered and Unordered Lists

An ordered list is simply a list that is numbered: List's are opened with an opening tag, <ol>, each individual item is wrapped with <li> and </li> tags, and then the list is closed with another </ol> tag: 

<ol>
		     	<li>First List Item</li>
			 <li>Second List Item</li>
			<li>Third List Item</li>
</ol>

An unordered list contains bullet points rather than numbers. To make an unordered list you use <ul></ul> in place of <ol></ol>.

Div Tags

The <div> </div> ('division') tag allows you to divide your webpage into different containers and sections. 

<div> </div>

Nesting Tags

Placing one HTML tag into another is called nesting. You simply place one link inside another as follows. For example; to make a picture take you to a site, you insert the Image tag inside the Hyperlink tag as follows: 

<a href="https://www.codecademy.com/"> <img src="https://imageURL"/> </a>

Parent-Child Relationships

In an HTML nesting parent/child relationship; Web developers refer to the enclosing element as the parent element and the enclosed elements as children. We will discuss this relationship more when we talk about CSS
