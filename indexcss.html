Cascading Style Sheets

Cascading Style Sheets (CSS) is a style sheet language that describes how an HTML files should look/appear to the user. CSS was first proposed in October 1994 by Håkon Wium Lie, and was published two years later in December 1996.

Connecting CSS to HTML

CSS files are linked to HTML files using the HTML Link Element. You do this by putting a <link> tag between the <head></head> tags of your HTML page. Your <link> must include these three attributes:

  • A type attribute that should always be equal to "text/css". This tells the computer that you are linking your html file to a css file
  • The rel attribute specifies the relationship between the current document and the linked document. Your css is the html file's "stylesheet"
  • A href attribute that should point to the web address of your CSS file

<link type="text/css" rel="stylesheet" href="stylesheet.css"/>

CSS Syntax

  1. Selector: Any HTML element that you intend to style, such as <p>, <img>, or <div>
  2. Property: An aspect (color, size, type, etc.) of a selector.
  3. Value: A possible property setting. (a specified color, font, size, etc.) Each property-value is ended with a semicolon (;)

Hexadecimal Values

CSS comes equipped with over 140 named colours, such as red or blue. In addition to those basic colors, Developers can be even more specific in their color choices using Hexidecimal Values: Hexadecimal counting is base-16. Each digit can be the numbers 0 through 9 or the letters a through f. Hex values always start with a pound sign (#), are up to six "digits" long, and are case-insensitive.

Each Hexadecimal value represents mixtures of red, green and blue. 

#RRGGBB
  Red: #FF0000

You can simply search for "hex color palette" or "hex color picker" to find a bunch of options. I used this one!

RGB Colors

RGB (Red, Green, Blue) colors, similar to Hexidecimal colors, are created by three numbers (up to 256) representing red, green, and blue. When mixed together, the three values create a specific color. 

 Purple: rgb(128,0,128)

Sizing in CSS

Pixels (px) are dots on your computer screen. Specifying font sizes in pixels allows you to see exactly on their screen what you created on yours, which can be problematic as it assumes your screens are of similar size when they may not be.

CSS Properties

Border: The border property creates a border around the selected HTML Element, and supports several values. For example, for a border 2 pixels thick, solid, and red, you would type;

 selector {
				border 2px red solid;  
	}

Font-Family: Allows developer to change font of text. You can place mutiple font families as backup, the computer will default to the first, and the others will be be used as backup

 selector {
				    font-family: papyrus, Courier;  
	}

Font-size: Allows developer to change size of font. Meaured in pixels or ems

 selector {
				font-size: 20px;  
	}

Nested Selectors: You can apply a value to certain CSS elements by listing everything it is in:

 div div p {
				property: value;  
	}

Universal Selector (*): Allows you to apply a certain CSS to every element on the page:

 * {
				property: value;  
	}

Direct Children: You can grab direct children (an element that is directly nested inside another element with no elements in between) by using the > symbol:

	div > p {
				property: value;  
	}

Classes: allow you to take elements of different types and give them the same styling. Classes are assigned to HTML elements with the word class and an equals sign:

        <div class="class"></div> 
	<img class="class"/>

Classes are identified in CSS with a dot (.):

	.class {
				property: value;  
	}

IDs: allow you to apply style to a single instance of a selector, rather than all instances. Id’s are assigned to HTML elements with the word id and an equals sign:

<div id="first"></div>

IDs are identified in CSS with a pound sign (#):

	#first {
				property: value;  
	}

Pseudo-Class Selectors: allow you to control the appearance of unvisited and visited links:

  • a:link: An unvisited link
  • a:visited: A visited link
  • a:hover: A link you're hovering your mouse over

	selector:pseudo-class_selector {
				property: value;  
	}

First-Child: is used to apply styling to only the elements that are the first children of their parents:

	p:first-child(2) {
				property: value;  
	}

Nth-Child: is used to apply styling to any child of an element after the first child; you just add the child's number in parentheses after the pseudo-class selector: 

	p:nth-child(2) {
				property: value;  
	}

Positioning HTML Elements With CSS

Each HTML element is like a tiny box or container that holds the pictures and text you specify.

  • block: This makes the element a block box. It won't let anything sit next to it on the page! It takes up the full width.
  • inline: This makes the element sit on the same line as another element, but without formatting it like a block. It only takes up as much width as it needs (not the whole line).
  • none: This makes the element and its content disappear from the page entirely

The margin is the space around an element. We can adjust the margin to move our HTML elements closer to or farther from each other. If you want to specify a particular margin, you can do it like this: 

	selector {
				margin-top: value;  
				margin-right: value;
				margin-bottom: value;
				margin-left: value;
	}

Margins can also be set all at once: you just start from the top margin and go around clockwise (going from top to right to bottom to left). For instance: margin: 1px 2px 3px 4px;

The padding is the spacing between the content and the border. We can adjust this value with CSS to move the border closer to or farther from the content. It is set similarily to the margins, individually or all at once

The border is the edge of the element. It's what we've been making visible every time we set the border property.

The content is the actual "stuff" in the box. In a <p> element, the "stuff" is the text of the paragraph.

Floating

Floating an element allows you to stack elements

  • clear:left; will immediately move element below any floating elements on the left side of the page.
  • clear:right; will move element below any floating elements on the right side of the page.
  • clear: both; will get out of the way of elements floating on the left and right.

	selector {
			clear: right, left, or both;
	}

Z-Index

The Z-index: a measure of importance: the higher an element's z-index, the higher it will be "stacked" on the page. Just set z-index to the number of importance.

Introduction to SASS

SASS Stands for Syntactically Awesome StyleSheets. SASS is an extension language (programming language interpreter that broadens the original application) for CSS. Sass allows you to write clean, sustainable CSS code and solve common repetition and maintenance challenges present in traditional CSS.

Sass can't be directly interpreted by your browser, so it must first be compiled (converted to lower level code so that it can be executed), to CSS before the browser can directly understand it.

Start using SASS

There are two main ways to start using SASS, you can either download a coding application that uses SASS (such as LiveReload, Hammer, or CodeKit), or you can install SASS using the command line

To install SASS, you will first need to install Ruby, as SASS has a Ruby Dependency. You can install Ruby here if you are on Windows, but if you have a mac it comes pre-installed.

  1. Open your Terminal or Command Prompt. On the Mac the Terminal.app comes installed by default. It's located in your "Utilities" folder. On Windows, run `cmd`.
  2. Install Sass. Ruby uses Gems to manage its various packages of code like Sass. In your open terminal window type:

    3. gem install sass

Preprocessing

SASS Allows you to use features that don't exist in CSS yet like variables (allow you to store information that you want to reuse throughout your stylesheet), nesting (makes code more readable), mixins (lets you make groups of CSS declarations that you can reuse throughout your site), and inheritance (lets you share a set of CSS properties from one selector to another) among other things.

SASS is simply a more advanced, professional, style of CSS. Visit their website here for more information
